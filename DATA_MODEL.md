# Growth Atlas 第一版数据模型

## 1. 文档目的

本文档基于 `README.md`、`MVP_SPEC.md`、`USER_FLOW.md` 和 `AI_CONVERSATION.md`，定义 Growth Atlas MVP 阶段需要保存的核心数据、各数据模块的职责及其关系。

本文档只描述产品层的数据结构，不涉及数据库选型、表结构代码、接口或具体实现。

Growth Atlas 的数据模型服务于以下成长闭环：

**反思（Reflection）→ 洞察（Insight）→ 决策（Decision）→ 成长（Growth）**

设计原则是：只保存完成 MVP 核心体验所必需的信息；区分用户确认的信息与 AI 推测；长期记忆由用户自主决定；所有成长判断都应可追溯、可修正、可删除。

## 2. MVP 需要保存的核心数据

Growth Atlas 第一版需要保存六类核心数据：

1. **User（用户）**：身份、基础设置以及隐私和记忆偏好。
2. **Personal Profile（个人成长档案）**：经用户确认、可用于个性化分析的长期背景。
3. **Reflection（日常反思记录）**：一次困惑从问题、分析、行动到回顾的完整成长记录。
4. **Conversation（AI 对话记录）**：一次结构化对话的过程、消息与状态。
5. **Growth Score（成长地图评分）**：基于可追溯证据形成的六维度阶段性评估。
6. **Memory（长期记忆）**：从档案或成长记录中提取、经用户明确同意后可跨对话使用的信息。

其中，Conversation 是过程数据，Reflection 是用户主动保存后的成长成果；Personal Profile 与 Memory 是长期个性化背景；Growth Score 是阶段性总结，不是人格诊断或精确能力测验。

## 3. 通用数据规则

六类数据应尽量采用一致的通用信息：

- **唯一标识**：用于区分每一条数据。
- **所属用户**：明确数据归属于哪位用户。
- **创建时间与更新时间**：支持时间顺序、变化追踪和回顾。
- **状态**：区分草稿、待确认、已确认、已归档或已删除等状态。
- **来源**：区分用户直接输入、用户确认、AI 生成、历史记录归纳。
- **可信度**：区分事实、用户解释、历史观察和 AI 待验证推测。
- **用户确认状态**：明确内容是否已经由用户审阅或同意。
- **删除状态**：支持用户删除及后续清理，不以“隐藏”代替删除。

通用可信度建议分为四级：

1. 当前对话中用户明确确认的信息。
2. 用户确认的个人成长档案信息。
3. 用户主动保存且完成回顾的历史成长记录。
4. AI 概括或推测，仅作为待验证线索。

第四级信息不得自动升级为长期档案或确定的用户特征。

## 4. User（用户）

### 4.1 模块作用

User 保存账号身份与产品偏好，是其他所有数据的归属起点。该模块不承担详细成长画像，避免身份信息与敏感成长内容过度混合。

### 4.2 MVP 数据结构

| 信息组 | 保存内容 | 说明 |
| --- | --- | --- |
| 身份信息 | 用户标识、登录标识 | 只保存登录和识别所必需的信息 |
| 基础设置 | 昵称、界面语言、时区 | 昵称可为空，不要求真实姓名 |
| 使用状态 | 注册时间、最近使用时间、账号状态 | 支持账号管理与数据生命周期 |
| 隐私确认 | 隐私说明版本、同意状态、同意时间 | 记录用户同意了什么版本的规则 |
| 保存偏好 | 是否默认询问保存 Reflection、是否保留完整对话 | MVP 建议默认先询问，不默认长期保存 |
| 记忆偏好 | 长期记忆总开关、是否允许 AI 提议新记忆 | 关闭后不得创建或调用长期记忆 |

### 4.3 边界

- 不强制保存真实姓名、学校、精确住址等非必要身份信息。
- 账号同意不等于同意保存每一条长期记忆；长期记忆仍需单独确认。
- 用户关闭记忆后，历史 Memory 不应继续用于新对话；用户可以选择保留、导出或删除它们。

## 5. Personal Profile（个人成长档案）

### 5.1 模块作用

Personal Profile 保存用户在首次引导或后续主动编辑中确认的长期背景，用于减少重复询问，并使 AI 将建议与用户的目标、价值观和现实条件联系起来。

### 5.2 MVP 数据结构

| 信息组 | 保存内容 | 说明 |
| --- | --- | --- |
| 档案状态 | 草稿、待确认、已确认、已归档 | 只有已确认内容可作为稳定个性化依据 |
| 基本背景 | 年龄段、学习或工作阶段、用户愿意提供的背景 | 避免收集与成长分析无关的敏感细节 |
| 当前阶段 | 当前人生阶段、主要情境 | 例如升学、毕业、专业选择等 |
| 压力来源 | 当前主要压力及用户描述 | 不将其转换为心理诊断 |
| 近期目标 | 当前希望达成的目标、时间范围 | 允许用户标记已完成或已失效 |
| 长期方向 | 较长期的方向与期待 | 可为空，避免强迫用户过早确定人生方向 |
| 重要价值观 | 用户明确表达并确认的重要价值 | 保留用户原意，不自动贴标签 |
| 自我描述 | 经用户确认的思维习惯、行为倾向和个人特点 | 必须标明是用户自述或经用户确认的总结 |
| 版本信息 | 当前版本、确认时间、上次复核时间 | 支持档案随时间变化，不覆盖历史语境 |

### 5.3 更新规则

- AI 可以提出档案更新建议，但不能因一次对话自动改写档案。
- 新目标、新特征或模式总结必须由用户确认后生效。
- 当前表述与旧档案冲突时，以当前用户说明为优先，并询问是否更新档案。
- 用户可以查看、纠正、补充、停用或删除档案内容。

## 6. Reflection（日常反思记录）

### 6.1 模块作用

Reflection 是 Growth Atlas 的核心成长记录。它承载一次真实困惑经过结构化分析、用户决策、行动和后续回顾形成的闭环。只有用户主动选择“保存为成长记录”后，才成为正式 Reflection。

### 6.2 MVP 数据结构

| 信息组 | 保存内容 | 说明 |
| --- | --- | --- |
| 基础信息 | 标题或问题摘要、创建时间、更新时间、记录状态 | 标题可由用户修改 |
| 核心问题 | 本次希望解决的一个问题、期望结果 | 避免一次记录混合过多主题 |
| 已确认事实 | 事件、行为、条件、限制 | 与解释和推测分开保存 |
| 未知信息 | 尚未确认或仍需验证的信息 | 防止 AI 把缺失信息补成事实 |
| 情绪与诉求 | 主要感受、触发点、强度、希望获得或保护的内容 | 属于敏感内容，应遵循最少保存原则 |
| 认知与洞察 | 用户当前判断、关键假设、矛盾、替代视角、个人洞察 | 标注用户确认内容与 AI 待验证推测 |
| 方案与权衡 | 主要选项、收益、代价、风险、可逆性、所需条件 | 用于支持而非替代用户决策 |
| AI 建议 | AI 建议方向及依据 | 明确不是用户最终决定 |
| 用户行动 | 用户确认的下一步、执行时间或情境、最低完成标准、障碍与备用方案 | 只有用户确认后才作为正式决策保存 |
| 后续回顾 | 实际行动、结果、感受变化、被验证或推翻的假设、可复用认识 | 可在行动后持续补充 |
| 来源关联 | 对应 Conversation、引用的 Profile 或 Memory | 保证分析可追溯 |

### 6.3 状态建议

- **草稿**：对话仍在进行，尚未形成正式成长记录。
- **待确认**：AI 已生成总结，等待用户修正和确认。
- **已保存**：用户主动保存为成长记录。
- **已回顾**：用户补充了实际行动和结果。
- **已归档或已删除**：不再作为活跃记录，或按用户要求清理。

### 6.4 保存边界

MVP 正式 Reflection 至少保存：问题或事件摘要、关键分析与个人洞察、用户确认的行动方案、用户后续补充的结果或变化、创建与更新时间。

未完成或未确认的 AI 推测可以在当前对话中暂存，但不得作为确定的用户特征写入 Personal Profile 或 Memory。

## 7. Conversation（AI 对话记录）

### 7.1 模块作用

Conversation 保存一次 AI 结构化追问的过程，使用户能够继续未完成的对话，并为最终 Reflection 提供来源。它不等于长期成长记忆，也不应因为被记录就自动用于未来个性化。

### 7.2 MVP 数据结构

| 信息组 | 保存内容 | 说明 |
| --- | --- | --- |
| 会话信息 | 会话标识、标题、开始时间、最后更新时间、状态 | 状态可为进行中、已完成、中止、已删除 |
| 会话类型 | 首次档案建立、日常反思、后续回顾 | 决定对话目标与输出类型 |
| 当前阶段 | 聚焦问题、事实、情绪、认知、决策、成长、报告确认 | 支持继续对话和流程判断 |
| 消息记录 | 消息角色、内容、时间、顺序 | 角色至少区分用户、AI、系统提示 |
| 内容标记 | 事实、解释、推测、建议、用户确认 | 支持可信度和后续提取 |
| 安全状态 | 是否触发安全边界、处理状态 | 仅记录实现安全流程所需的最少信息 |
| 关联信息 | 对应 Profile、Reflection、被引用的 Memory | 记录实际使用了哪些长期信息 |
| 保存策略 | 临时或长期保留、保留期限、用户选择 | 对话保存与成长记录保存应分开选择 |

### 7.3 保存规则

- 用户可以保存 Reflection，而不必永久保留完整逐字对话。
- 若用户不选择长期保留完整对话，可只保存完成流程所必需的摘要和确认结果，并按明确期限清理原始消息。
- Conversation 中出现的信息不会自动成为 Memory。
- 对话被删除后，由它单独产生且无其他有效来源的长期记忆应提示用户一并复核或删除。

## 8. Growth Score（成长地图评分）

### 8.1 模块作用

Growth Score 用于表达用户在六个成长维度上的阶段性状态和变化：

1. 自我认知。
2. 情绪管理。
3. 决策能力。
4. 行动力。
5. 人际关系。
6. 学习成长。

这里的“评分”不是心理测验、能力排名或人格诊断。MVP 应优先展示有依据的定性评估和变化趋势；信息不足时明确显示“信息不足”，不生成伪精确数字。

### 8.2 MVP 数据结构

| 信息组 | 保存内容 | 说明 |
| --- | --- | --- |
| 评估周期 | 周期开始、周期结束、生成时间 | 说明判断所覆盖的时间范围 |
| 维度 | 六个成长维度之一 | 每个周期最多六条维度结果 |
| 状态 | 信息不足、初步观察、有证据支持 | 表示证据充分程度，不表示人格高低 |
| 趋势 | 暂无趋势、改善、稳定、波动、需要关注 | 必须由跨记录证据支持 |
| 已有表现 | 有记录支持的行为、认识或能力 | 使用可观察描述 |
| 观察到的变化 | 与更早记录相比的变化 | 无历史基线时为空 |
| 关注模式 | 重复出现且影响目标的模式 | 使用试探性语言，避免绝对结论 |
| 下一阶段建议 | 一项或少量可观察的成长重点 | 与用户当前目标和资源相匹配 |
| 证据引用 | 支持结论的 Profile、Reflection 或回顾记录 | 每项判断都应可追溯 |
| 置信说明 | 证据数量、信息缺口、AI 推测说明 | 帮助用户理解结论边界 |
| 用户反馈 | 同意、部分同意、不同意及修正内容 | 用户修正应影响后续使用 |

### 8.3 MVP 展示与生成规则

- 第一版不要求使用 0—100 等数值分数；建议用“状态 + 趋势 + 证据”表达。
- 如果产品界面必须展示数值，数值也必须附带证据、置信说明和“非诊断”提示，且不得进行用户间排名。
- 记录不足时，状态只能是“信息不足”或“初步观察”。
- 评分只基于用户确认的档案、主动保存的 Reflection 和用户补充的行动结果。
- AI 未确认推测不能单独支撑评分或趋势。
- 用户可以否定或修正某个维度结论；被否定内容不得继续作为确定事实引用。

## 9. Memory（长期记忆）

### 9.1 模块作用

Memory 保存用户允许 AI 在未来对话中再次使用的少量长期信息。它不是完整对话副本，也不是系统自动建立的隐性人格画像。

Personal Profile 是用户主动维护的结构化档案；Memory 是为了跨对话复用而从档案或历史记录中选出的具体信息。两者可以关联，但不能互相默认替代。

### 9.2 MVP 数据结构

| 信息组 | 保存内容 | 说明 |
| --- | --- | --- |
| 记忆内容 | 一条清晰、简短、可独立理解的信息 | 一条 Memory 只表达一个主题 |
| 记忆类型 | 背景、目标、价值观、偏好、已验证模式、有效策略 | 不保存诊断性标签 |
| 来源 | Profile、Reflection、用户直接创建 | 必须能回到原始来源 |
| 来源语境 | 产生该记忆的主题和时间 | 避免脱离语境使用旧信息 |
| 提出者 | 用户主动添加或 AI 建议 | AI 建议不等于已经保存 |
| 确认状态 | 待确认、已同意、已拒绝 | 只有已同意可跨对话调用 |
| 适用范围 | 所有成长对话或限定主题 | 让用户控制记忆被用于哪里 |
| 有效期 | 长期、截至某日、需定期复核 | 目标和阶段性背景不应永久有效 |
| 敏感级别 | 一般、敏感 | 敏感记忆需要更明确的提示和控制 |
| 使用记录 | 最近使用时间、被哪些会话引用 | 让长期记忆的使用可解释、可审计 |
| 生命周期 | 创建、确认、复核、停用、删除时间 | 支持用户完整控制 |

### 9.3 记忆写入流程

1. 用户主动要求记住，或 AI 提议一条可能有长期价值的信息。
2. 系统向用户展示将保存的具体内容、来源和未来用途。
3. 用户选择同意、修改后同意或拒绝。
4. 只有明确同意后，Memory 才进入可用状态。
5. 未来引用时说明来源语境，并在可能过期时先询问是否仍然有效。

### 9.4 记忆自主权

- 用户可以查看“AI 记住了什么”。
- 用户可以逐条编辑、停用、删除或限制使用范围。
- 用户可以关闭长期记忆总开关。
- 拒绝保存某条记忆不影响用户继续完成当前对话。
- 删除来源数据时，系统应提示并处理关联 Memory，避免保留失去依据的画像。
- 不把一次情绪、一次失败或少量记录概括为永久人格特征。

## 10. 数据之间的关系

### 10.1 关系概览

| 主体 | 关系 | 目标数据 |
| --- | --- | --- |
| User | 一对零或一 | Personal Profile |
| User | 一对多 | Conversation |
| User | 一对多 | Reflection |
| User | 一对多 | Growth Score 评估周期 |
| User | 一对多 | Memory |
| Conversation | 零或一对一 | Reflection |
| Reflection | 可关联一个 | 来源 Conversation |
| Reflection | 可产生零到多条 | Memory 建议 |
| Personal Profile | 可产生零到多条 | Memory |
| Growth Score | 多对多引用 | Personal Profile 与 Reflection |
| Conversation | 多对多引用 | Memory |

### 10.2 关系说明

- 一个 User 在 MVP 中拥有一份当前 Personal Profile；档案可以有历史版本。
- 一个 User 可以发起多次 Conversation，但并非每次对话都会保存为 Reflection。
- 一次聚焦同一问题的 Conversation 最多形成一条主要 Reflection；后续回顾可以继续更新该 Reflection，或由新的回顾会话关联它。
- Reflection 可以引用当时使用的 Profile 和 Memory，以说明分析依据。
- Memory 可以来源于 Profile 或 Reflection，但必须经过独立确认。
- Growth Score 按阶段生成，每个阶段包含六个维度结果，并引用支撑结论的档案与 Reflection。
- 删除或修正来源数据后，依赖它的 Memory 和 Growth Score 应被标记为需要复核，而不是继续无条件使用旧结论。

## 11. 隐私、安全与数据自主原则

### 11.1 最少保存

- 只收集完成成长分析、回顾和个性化所需的数据。
- 正式成长记录优先保存结构化摘要，不默认永久保存全部逐字对话。
- 对情绪、关系、压力等敏感内容，允许用户不填、修改或删除。

### 11.2 明确同意

- 首次使用时说明会保存哪些数据以及保存目的。
- 保存 Reflection、保留完整 Conversation、创建 Memory 是不同选择，不能合并为一次模糊授权。
- 隐私说明变更后，应记录新版本并在必要时重新征得同意。

### 11.3 用户控制

- 用户可以查看、纠正、导出和删除自己的数据。
- 用户可以决定是否保存一次成长记录、是否保留完整对话、是否启用长期记忆。
- 用户当前的明确说明优先于历史档案和旧记忆。
- 用户删除账号时，应支持删除其 Profile、Reflection、Conversation、Growth Score 和 Memory。

### 11.4 AI 推测边界

- AI 生成内容必须与用户确认内容分开标记。
- AI 推测只能作为待验证线索，不能自动成为档案、记忆或评分依据。
- 不生成心理诊断、人格定性或缺乏证据的确定性人生结论。
- 成长结论应能追溯到用户确认的数据，并明确证据不足之处。

### 11.5 敏感与安全信息

- 涉及紧急危险时，数据记录应以支持当下安全流程所需的最少信息为限。
- 不为成长画像目的长期沉淀危机表达、医疗、法律或其他高风险敏感信息。
- 对敏感数据应采用更严格的访问控制、传输与存储保护，并记录必要的访问和使用情况。

## 12. MVP 范围结论

第一版数据模型应优先支撑以下闭环：

1. 用户建立并确认个人成长档案。
2. 用户通过一次 AI 对话梳理真实困惑。
3. 用户确认行动方案并主动保存为 Reflection。
4. 用户在行动后补充结果与变化。
5. 系统基于可追溯记录形成六维度阶段性 Growth Score。
6. 用户自主选择哪些信息成为可跨对话使用的 Memory。

MVP 暂不需要复杂标签体系、知识图谱、社交关系、排行榜、游戏化积分或精确人格评分。数据结构的首要目标是让成长轨迹可回顾、分析依据可追溯、AI 记忆可控制。
